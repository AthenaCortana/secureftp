Qu'est ce que SSL?

SSL, ou Secure Sockets Layer, est une technologie sécurisée élaborée par Netscape Communications afin de permettre le chiffrement et l'authentification des communications. C'est dans les communications entre les navigateurs et les serveurs Web que l'utilisation de SSL est la plus visible (quand les URLs qui commencent par https indiquent qu'une connexion SSL va être utilisée).

Secure FTP étend cette technologie pour permettre le transfert de fichiers sur une connexion sécurisée (sous réserve que le serveur FTP supporte les connexions SSL).

Qu'est ce qui est supporté?

Secure FTP propose deux options de connexion SSL: connexions implicites et explicites.

SSL implicite

La sécurité implicite (dite aussi sécurité passive), débute par une connexion SSL dès que la connexion est établie. Avec la sécurité implicite, le serveur définit un port différent (en général le port 990), que le client FTP devra utiliser pour la connexion sécurisée. Note: SSL implicite fut initialement défini dans plusieurs ébauches du protocole FTPS mais n'est plus recommandé.

SSL explicite

La sécurité explicite (dite aussi sécurité active), débute par une connexion FTP classique (non sécurisée) au cours de laquelle une commande spéciale envoyée au serveur, AUTH SSL, lui demande de passer en mode sécurisé. Les spécifications sont formellement définies dans la RFC 2228.

Certificats clients

L'authentification client SSL peut aussi être utilisée pour authentifier le client auprès du serveur. Cette forme d'authentification n'est pas effectuée pendant la phase de login, mais au contraire quand la connexion sécurisée initiale est établie.

Secure FTP propose deux méthodes pour construire et envoyer un certificat client:
  1. Générer un certificat auto-signé. Ce certificat, valide pendant 3 ans, peut être généré en répondant à quelques questions simples.

  2. Utiliser une paire clé/certificat fournie par un administrateur. Si votre entreprise possède sa propre Autorité de Certification (CA), vous pouvez importer la clé privée et le certificat public dans le client. La clé privée doit être soit au format enveloppe PKCS#12, soit isolée au format PKCS#8. Le certificat de l'Autorité de certification et le certificat public doivent être soit au format enveloppe PKCS#12, soit isolés au format PEM ou DER.

  3. Si la paire clé/certificat est protégée par un mot de passe, inscrivez ce mot de passe dans le champ approprié.
Vous trouverez ces options dans la rubrique de menu Aide > Gérer les certificats.
