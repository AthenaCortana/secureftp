Comment se connecter via la boîte de dialogue "Ouvrir une connexion"


Se connecter à un serveur est une opération très simple. Vous devez seulement connaître quelques informations de base, comme le nom d'hôte, le nom d'utilisateur et le mot de passe.

Le Nom d'hôte est le nom de la machine à laquelle vous voulez vous connecter. Le Nom d'utilisateur est le nom que vous utiliseriez pour vous identifier auprès de cette machine. Le Mot de passe est la clé qui, associée au Nom d'utilisateur, vous permet de vous authentifier. Dans certains cas, vous pourrez vous connecter en mode Anonyme, auquel cas vous aurez des privilèges moins étendus qu'un utilisateur standard.

Par défaut, SecureFTP tentera d'établir une connexion SSL explicite. Si cette tentative échoue, vous en serez averti(e) et il vous sera proposé d'établir une connexion non sécurisée. Note: Vous devez cependant accepter cette dernière option avec la plus grande prudence. En effet, dans ce cas, votre mot de passe et vos données seront transmis en clair sur le réseau.

Dans la boîte de dialogue "Ouvrir une connexion", vous pouvez aussi définir un répertoire de démarrage distant et un répertoire de démarrage local, dans lesquels le programme se positionnera en cas d'authentification réussie.

Vous pouvez aussi définir le Type de connexion. Par défaut le type de connexion est passif, ce qui devrait convenir à la plupart des utilisateurs. Le type de connexion passif implique que lorsqu'un transfert de données est effectué, le serveur décidera où vous allez vous connecter. Si vous êtes NATé(e) derrière un pare-feu, vous devrez utiliser ce type de connexion passif, ou, si le serveur le supporte et si une connexion SSL explicite est établie, sélectionnez l'option Canal de commande en clair et désélectionnez l'option Type de connexion passif.

L'option Canal de commande en clair n'est disponible que lorsqu'une connexion SSL explicite est établie. Quand l'utilisateur a été authentifié de manière sécurisée, le canal de commande (ou de contrôle) revient à une connexion en clair. Cette option permet aux utilisateurs (NATés) derrière un pare-feu de d'utiliser les connexions de type actif (ou PORT).

Si vous enregistrez ces options dans vos signets, vous pourrez les retrouver via le menu "Signets" ou l'option Profil de la boîte de dialogue "Ouvrir une connexion". Pour plus d'information à propos des signets : ici.
