How To Connect Via The Open Connection Dialog


Connecting to a server is a pretty straightforward process. At the bare minimum, you will need to know some basic information, including the host name, user name and password.

The host name is the name of the machine you wish to connect. The user name is the name you would use to log in to that machine. The password is the key that allows your user name in. In some cases you may log in anonymously, in which case you may have less privileges than a standard user.

By default, we will attempt to make an explicit SSL connection. If this connection fails, we will notify you and offer to make an insecure connection. Note: You should choose this option carefully. If you allow for this, your password and data are open to the public.

In the Open Connection dialog you can also specify a starting remote folder and local folder, which the program will navigate to upon successful login.

You can also set the connection type. The default is a passive connection type, which for most users will be sufficient. What this means is when a data transfer is made, the server will decide where you will connect to. If you are behind a NAT firewall, you will want this option selected or, if the server supports it and an explicit SSL connection is made, select the Clear Command Channel option and the passive option deselected.

The Clear Command Channel option is available only when an explicit SSL connection is made. After the user has been securely authenticated, the command (or control) channel is reverted back to a clear connection. This option allows users behind a (NAT) firewall the ability to get active (or PORT) data connections to work.

If you save these options to your bookmarks you can access them via the Bookmarks menu or the Profile option in the Open Connection dialog. You can get more information about bookmarks here.
