O Que É SSL?

SSL, sigla inglesa de Secure Sockets Layer, é uma tecnologia segura desenhada pela Netscape Communications para permitir comunicação autenticada encriptada. A SSL é mais visivelmente utilizada em comunicações entre Navegadores da Web e servidores Web (onde os URLs, começando por https, indicam que está a ser usada uma ligação SSL).

O Secure FTP estende esta mesma tecnologia para permitir a transferência de ficheiros através de uma ligação segura (desde que o servidor de FTP suporte ligações SSL).

O Que É Suportado?

O Secure FTP oferece duas opções de ligação SSL: implícita e explícita.

SSL Implícita

Segurança implícita (também conhecida por segurança passiva), inicia uma ligação SSL logo que a ligação é estabelecida. Com segurança implícita, o servidor define uma porta diferente para o cliente FTP (usualmente a porta 990) a ser usada para a ligação segura. Nota: foi inicialmente definida SSL Implícita em vários projectos FTPS mas já não é recomendada.

SSL Explícita

Segurança explícita (também conhecida por segurança activa), inicia com uma ligação básica FTP (insegura) em que um comando especial, AUTH SSL, informa o servidor para transitar para modo seguro. A especificação é formalmente definida em RFC 2228.

Certificados de Cliente

Uma forma adicional de autenticar o cliente no servidor é usar autenticação SSL de cliente. Esta forma de autenticação não ocorre durante a entrada no sistema ("login"), mas sim quando a primeira ligação segura é feita.

O Secure FTP oferece duas formas de definir e enviar um certificado de cliente:
  1. Gerar um certificado auto-assinado. Este certificado pode ser gerado respondendo a algumas perguntas simples, e é válido por 3 anos.

  2. Use um par chave/certificado emitido por um administrador. Se a sua companhia tem o seu próprio Certificado de Autoridade, pode importar a chave privada e o certificado público para o cliente. A chave privada tem de estar numa PKCS#12 keystore, ou sozinha em formato PKCS#8. Os certificados público/CA têm de estar numa PKCS#12 keystore, ou sozinhos nos formatos PEM e DER.

  3. Se o par certificado/chave estiver protegido por uma frase-passe, introduza a frase-passe no campo próprio.
Estas opções podem ser encontradas no menu Ajuda > Gerir Certificados
