Perguntas Freqüentes

Posso recuperar arquivos apagados?

Numa palavra, não. Quando se estão eliminando arquivos do servidor remoto, não há caixote de lixo ou caixa de reciclagem, assim os arquivos são apagados imediatamente. Localmente, o seu sistema poderá suportar restauração de arquivos, mas dado que o Secure FTP é independente da plataforma, não temos hipótese de fornecer essa funcionalidade.

Suporte Para NAT

NAT, ou "Network Address Translation" (que significa Tradução de Endereço de Rede), está a tornar-se mais frequentemente usada. Se estiver a utilizar uma barreira de proteção ("firewall") ou outro dispositivo que lhe dá vários IPs a partir de um, a hipótese é estar usando NAT. A única coisa que tem de observar, quando estiver usando NAT e o Secure FTP, é quando não estiver usando conexão do tipo passivo. Se estiver por trás de NAT e está a efetuar uma conexão segura, terá de fazer uma de duas coisas: conexão passiva, ou, permitir a opção Canal de Comando Visível juntamente com uma conexão de dados (PORTA) ativa, de outro modo as suas transferências de dados irão falhar. Isso ocorre porque durante uma conexão FTP não-segura o seu dispositivo NAT realmente "escuta" os comandos FTP e faz o endereçamento correto. Mas, quando a conexão é segura, o dispositivo NAT não consegue penetrar e fazer o seu trabalho corretamente. Quando se fazem conexões passivas, o NAT não terá de se preocupar com este comportamento.
