Modos de Transferência

Existem três modos de transferência de dados: auto, binário e texto (ASCII). Os ficheiros são tratados de maneira diferente quando se trata de um documento de texto simples versus todos os outros tipos de ficheiros.

Quando se selecciona texto como modo de transferência, o servidor pode mudar os fins-de-linha do ficheiro. Ou seja, os caracteres de retorno ("Carriage returns, CRs") podem ser substituídos pelos caracteres de retorno do seu sistema. Isto permite obter ficheiros de texto correctamente formatados.

Porém, podem acontecer problemas se aplicar modo de texto a ficheiros binários (tais como documentos Word). Aqui, vai querer que o servidor não interprete nada no ficheiro e que o envie tal como está.

Se não desejar ter de mudar entre os modos de transferência, é melhor seleccionar o modo auto e deixar o Secure FTP decidir qual é o melhor.
