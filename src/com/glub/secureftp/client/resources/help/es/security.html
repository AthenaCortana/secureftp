¿Qué es SSL?

SSL, o Capa de Sockets Segura, es una tecnología segura diseñada por Netscape Communications para habilitar la comunicación encriptada y autenticada. SSL es visible cuando se usa en comunicaciones entre navegadores Web y servidores Web (las URLs que comienzan con https indican que la conexión que se va a usar es SSL).

Secure FTP amplía esta tecnología para permitir la transferencia de archivos mediante una conexión segura (el servidor FTP soporta conexiones SSL).

¿Qué está soportado?

Secure FTP ofrece dos opciones de conexión SSL: conexiones implícita y explícita.

SSL Implícito

La seguridad implícita (también conocida como seguridad pasiva), comienza con una conexión SSL en el momento en el que ésta se establece. Con seguridad implícita, el servidor define un puerto diferente para el cliente FTP (el puerto es normalmente el 990) para que se use en la conexión segura. Atención: El SSl implícito fue inicialmente definido en varias versiones de FTPs pero actualmente está en desuso.

SSL Explícito

La seguridad explícita (también conocida como seguridad activa), comienza con una conexion básica FTP (insegura) donde un comando especial, AUTH SSL, le indica al servidor que cambie a un modo seguro. La especificación se define formalmente en RFC 2228.

Certificados de Clientes

Un modo adicional de autenticar un cliente en un servidor es usando la autenticación cliente SSL. Esta forma de autenticación no ocurre durante el proceso de login, sino cuando se realiza la conexión segura inicial.

Secure FTP ofrece dos maneras de establecer y enviar un certificado de cliente:
  1. Generar un certificado de autofirma. Este certificado puede ser generado al responder a unas simples questiones, y es válido durante 3 años.

  2. Utilice un par clave/certificado establecido de un administrador. Si su compañía posee su propio Certificado de Autoridad, puede importar la clave privada y publicar el certificado en el cliente. La clave privada necesita estar en un almacén de claves PKCS#12, o sola en formato PKCS#8. Los certificados públicos/CA necesitan estar en un almacén de claves PKCS#12, o solos en los formatos PEM o DER.

  3. Si el par certificado/clave está protegido por una frase-contraseña, escriba esa frase-contraseña en el campo apropiado.
Estas opciones se pueden encontrar en el elemento del menú de Certificados de Gestión de Ayuda >
