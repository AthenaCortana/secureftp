Cómo conectar a través del Diálogo de Abrir Conexión


Conectar con un servidor es un proceso sencillo. Como mínimo, necesitará saber alguna información básica, incluyendo el nombre del host, el nombre de usuario y la contraseña.

El nombre del host es el nombre de la máquina a la que desea conectarse. El nombre de usuario es el nombre que usted usará para identificarse en esa máquina. La contraseña es la clave que le permite a su nombre de usuario identificarse. En algunos casos, usted podrá identificarse como anónimo, pero tendrá menos privilegios que un usuario estandar.

Por defecto, intentaremos realizar una conexión SSL explícita. Si esta conexión fallase, se lo notificaremos y le ofreceremos realizar una conexión no segura. Atención: Debería escoger esta opción cuidadosamente. Si lo permite, su contraseña y datos estarán abiertos al público.

En el diálogo de Abrir Conexión puede especificar un directorio remoto y un directorio local, a los que el programa accederá directamente tras la correcta identificación de usuario.

Del mismo modo, usted puede elegir el tipo de conexión. El tipo por defecto es un tipo de conexión pasiva, que es suficiente para la mayoría de los usuarios. Esto significa que cuando se hace una transferencia de datos, el servidor decide dónde se conectará. Si usted tiene un cortafuegos NAT, elegirá esta opción o, si el servidor lo soporta y realiza una conexión SSL explícita, seleccione la opción Canal de Comando Clear y deseleccione la opción pasiva.

La opción Canal de Comando Clear está disponible sólo cuando se realiza una conexión SSL explícita. Después de que el usuario haya sido autenticado en un modo seguro, el canal de comando (o control) se vuelve una conexión Clear. Esta opción permite a los usuarios con un cortafuegos (NAT) activar conexiones de datos (o PUERTOS) para trabajar.

Si guarda estas opciones en sus favoritos podrá acceder a ellos a través del menú de Favoritos o a través de la opcion Perfil del Diálogo Abrir Conexión. Puede obtener más información acerca de favoritos aquí.
