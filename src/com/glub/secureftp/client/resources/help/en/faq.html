Frequently Asked Questions

Can I restore deleted files?

In short, no. When deleting files from the remote server, there is no trashcan or recycle bin, so files are deleted immediately. Locally, your system might support restoring files, but since Secure FTP is platform independent there is no way we can provide such functionality.

Support for NAT

NAT, or Network Address Translation, is becoming more frequently used. If you are using a firewall or other device that gives you many IPs from one, chances are you are using NAT. The only thing you need to watch when using NAT and Secure FTP is when you are not using a passive connection type. If you are behind NAT and are making a secure connection, you will need to make either: a passive connection, or enable the Clear Command Channel option along with an active (PORT) data connection, otherwise your data transfers will fail. This is because during an insecure FTP connection your NAT device actually listens for FTP commands and does the correct address routing. But when the connection is secure, the NAT device cannot peek in and do the right thing. When making passive connections, NAT will not have to worry about this behavior.
